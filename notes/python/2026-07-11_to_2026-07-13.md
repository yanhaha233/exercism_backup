## 1. String processing

### `split()`

Use `str.split()` to split a string into words.

```python
text = "How can mirrors be real"
words = text.split()

# ["How", "can", "mirrors", "be", "real"]
```

A common transformation pipeline is:

```text
string
-> split into words
-> transform each word
-> join the words
```

### `capitalize()`

`str.capitalize()` uppercases the first character and lowercases the remaining characters.

```python
"hello".capitalize()
# "Hello"
```

Example:

```python
[word.capitalize() for word in string.split()]
```

### `join()`

`join()` belongs to the separator string, not to `list`.

Wrong:

```python
words.join(" ")
```

Correct:

```python
" ".join(words)
```

A useful way to remember it:

```python
"separator".join(strings)
```

Jaden Casing example:

```python
def to_jaden_case(string):
    return " ".join(
        word.capitalize()
        for word in string.split()
    )
```

## 3. Sequence repetition with `*`

Strings can be repeated using multiplication.

```python
"#" * 5
# "#####"
```

An important Python behavior:

```python
"#" * 0
"#" * -1
"#" * -100
```

All return:

```python
""
```

For sequence repetition, a repetition count less than or equal to zero produces an empty sequence.

This makes the Codewars `maskify` solution very compact:

```python
def maskify(cc):
    return "#" * (len(cc) - 4) + cc[-4:]
```

There is no need for a separate `if len(cc) < 4` condition because both operations already handle short strings correctly.

## 5. `all()` for checking multiple conditions


## 6. `set()` and uniqueness

A `set` stores unique hashable values.

```python
set("hello")
# {'h', 'e', 'l', 'o'}
```

Repeated values are removed.

This is useful for Isogram problems:

```python
letters = [char for char in text.lower() if char.isalpha()]

return len(letters) == len(set(letters))
```

Important distinction:

```python
{"a", "b"}
```

is a set.

But:

```python
{{"a", "b"}}
```

tries to put a set inside another set and raises:

```text
TypeError: unhashable type: 'set'
```

A mutable `set` is not hashable and therefore cannot be an element of another `set`.

## 8. ISBN-10 validation: validate format before calculation

A major lesson from the ISBN Verifier exercise:

> Do not silently ignore invalid input.

For example, this logic is dangerous:

```python
for char in isbn:
    if char.isdigit():
        isbn_list.append(char)
```

It ignores every invalid character.

Input such as:

```text
3-598-21508-?X
```

should be invalid, not treated as though `?` never existed.

A safer validation process is:

```text
1. Remove allowed separators (`-`)
2. Check the resulting length
3. Validate the first 9 characters
4. Validate the final character
5. Perform the checksum calculation
```

Example:

```python
isbn = isbn.replace("-", "")

if len(isbn) != 10:
    return False

if not isbn[:9].isdigit():
    return False

if not (isbn[-1].isdigit() or isbn[-1] == "X"):
    return False
```

Important edge cases:

- Hyphens are optional.
- ISBN input is not guaranteed to contain exactly three hyphens.
- `X` is only valid as the final character.
- Invalid characters must be rejected explicitly.
- Validate the structure before calculating the checksum.

## 9. Pangram and Isogram: map the problem to a data structure

### Pangram

Question:

> Does the sentence contain every alphabet letter?

Useful tools:

```python
in
all()
set()
lower()
```

Possible approaches:

```python
alphabet = "abcdefghijklmnopqrstuvwxyz"

return all(letter in sentence.lower() for letter in alphabet)
```

or:

```python
letters = {
    char
    for char in sentence.lower()
    if char.isalpha()
}

return len(letters) == 26
```

### Isogram

Question:

> Does any letter appear more than once?

Useful idea:

```text
number of letters == number of unique letters
```

Example:

```python
letters = [
    char
    for char in string.lower()
    if char.isalpha()
]

return len(letters) == len(set(letters))
```

The important lesson is not only memorizing methods.

The more useful skill is mapping a problem to a tool:

```text
check existence      -> in
check every condition -> all()
remove duplicates     -> set()
normalize case        -> lower()
```

## 10. Darts: compare squared distance

For a point `(x, y)`, the distance from the origin is:

```python
(x ** 2 + y ** 2) ** 0.5
```

But when the problem only asks whether the point is inside a radius, the square root is unnecessary.

Instead of:

```python
distance <= radius
```

compare:

```python
x ** 2 + y ** 2 <= radius ** 2
```

Example:

```python
distance_squared = x ** 2 + y ** 2

if distance_squared <= 1:
    return 10
elif distance_squared <= 25:
    return 5
elif distance_squared <= 100:
    return 1
return 0
```

Lesson:

> Avoid unnecessary calculation when an equivalent comparison exists.

## 11. Perfect Numbers: divisor enumeration and square-root optimization

A number's divisors can be found in pairs.

If:

```python
number % i == 0
```

then both:

```python
i
number // i
```

are divisors.

Therefore, it is enough to iterate to:

```python
int(number ** 0.5) + 1
```

Example structure:

```python
divisor_sum = 0

for i in range(1, int(number ** 0.5) + 1):
    if number % i == 0:
        other = number // i

        if i != number:
            divisor_sum += i

        if other != i and other != number:
            divisor_sum += other
```

Important edge cases:

- Do not include the number itself as a proper divisor.
- For perfect squares, do not add the square-root divisor twice.
- When handling a divisor pair, add `other`, not `i`, for the second divisor.

The useful debugging question is:

```text
What exact value should this branch add?
```

rather than only checking whether the branch executes.

## 12. Resistor Color: mapping data to indexes

The resistor color exercise uses the position of a color in a predefined list as its numeric value.

```python
COLORS = [
    "black",
    "brown",
    "red",
    "orange",
    "yellow",
    "green",
    "blue",
    "violet",
    "grey",
    "white",
]
```

Example:

```python
COLORS.index("red")
# 2
```

The first two colors form the significant digits:

```python
value = first * 10 + second
```

The third color is a power-of-ten multiplier:

```python
value = (first * 10 + second) * 10 ** third
```

Main lesson:

> When a fixed ordered mapping exists, the list index itself can represent the value.

## 13. List comprehensions and generator expressions

List comprehension:

```python
[
    word.capitalize()
    for word in string.split()
]
```

Generator expression:

```python
(
    word.capitalize()
    for word in string.split()
)
```

`join()` can consume an iterable of strings directly:

```python
" ".join(
    word.capitalize()
    for word in string.split()
)
```

Therefore, creating an intermediate list is not always necessary.

Current focus:

> First write the logic clearly, then consider whether an intermediate list is needed.

## 14. Edge-case thinking


A useful checklist:

```text

invalid characters
duplicate values
case differences
boundary values
```

The main progress in these exercises is moving from:


toward:

