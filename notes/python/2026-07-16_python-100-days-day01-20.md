## 1. Big Learning Map

The first 20 files form a clear path:

```text
environment and first program
-> variables, types, operators
-> branch and loop control flow
-> built-in data structures
-> functions and modules
-> higher-order functions, decorators, recursion
-> object-oriented programming
```

The important shift is:

```text
syntax
-> small logic
-> reusable functions
-> reusable objects
```

At the beginning, the focus is writing code that runs. Later, the focus becomes writing code that can be reused, extended, and understood.

Core ideas:


Important pattern:


Important takeaway:

```text

while is usually for unknown repetition controlled by a condition.
```


### Day 17: 函数高级应用

Core ideas:

```text
Decorators add behavior to an existing function.
Decorators are higher-order functions.
Recursion lets a function call itself.
```

Decorator pattern:

```python
def record_time(func):
    def wrapper(*args, **kwargs):
        result = func(*args, **kwargs)
        return result

    return wrapper
```

Important decorator takeaway:

```text
@record_time
def download(...):
    ...

is equivalent to:

download = record_time(download)
```

Recursion checklist:

```text
1. Is the problem recursively defined?
2. What is the base case?
3. What is the recursive formula?
4. Will it converge quickly enough?
```

Important takeaway:

```text
Recursion without a clear base case or quick convergence can cause RecursionError.
For Fibonacci, plain recursion repeats too much work; iteration or caching is better.
```

### Day 18: 面向对象编程入门

Core ideas:

```text
Object-oriented programming models the world through objects.
Classes are abstract.
Objects are concrete instances.
Objects receive messages through methods.
```

Important pattern:

```python
class Student:
    def __init__(self, name, age):
        self.name = name
        self.age = age

    def study(self, course_name):
        print(f"{self.name} is studying {course_name}")
```

Important takeaway:

```text
Class design is abstraction:
attributes are data abstraction;
methods are behavior abstraction.
```

### Day 19: 面向对象编程进阶

Core ideas:

```text
Python objects can dynamically receive new attributes.
__slots__ can restrict allowed attributes.
@property turns method-style calculations into attribute-style access.
@staticmethod and @classmethod define messages sent to the class rather than a normal object.
Inheritance reuses code from existing classes.
Polymorphism means the same method call can behave differently on different subclasses.
```

Important patterns:

```python
class Triangle:
    @staticmethod
    def is_valid(a, b, c):
        return a + b > c and b + c > a and a + c > b

    @property
    def perimeter(self):
        return self.a + self.b + self.c
```

Important takeaway:

```text
OOP is not just class syntax.
The real value is managing shared behavior, specialization, and extension.
```

## 3. Data Structure Selection Guide

Choose a container based on the problem:

```text
Need ordered and mutable data:
list

Need ordered but immutable grouped data:
tuple

Need text:
str

Need uniqueness or set operations:
set

Need lookup by key:
dict
```

Common mental checks:

```text
Do I care about order?
Do I need to modify the data?
Do I need fast lookup?
Do I need to remove duplicates?
Do I need key-value relationships?
```

## 4. Function Thinking

The function chapters are the turning point from beginner syntax to reusable code.

A function is useful when:

```text
the code repeats;
the behavior has a clear name;
the logic can be tested independently;
the caller should not care about internal details.
```

A good function usually has:

```text
clear input
clear output
one stable responsibility
limited side effects
```

Important distinction:

```text
Passing a function:
calc(0, add, 1, 2, 3)

Calling a function:
add(1, 2)
```

This distinction also helps understand decorators:

```text
return wrapper
returns the function object

return wrapper()
calls the function immediately
```

## 5. Object-Oriented Thinking

The OOP chapters emphasize that object-oriented programming is a modeling method.

Useful modeling process:

```text
1. Read the requirement.
2. Mark important nouns.
3. Decide which nouns should become classes.
4. Decide which data belongs to each class.
5. Mark important verbs.
6. Turn those verbs into methods.
7. Decide relationships between classes.
```

Examples:

```text
Poker has cards.
Player has cards.
Player gets one card.
Player arranges cards.
Employee has name.
Manager is an Employee.
Programmer is an Employee.
Salesman is an Employee.
Each Employee can calculate salary, but the formula differs.
```

The salary example is a good first model of polymorphism:

```text
same call:
emp.get_salary()

different object types:
Manager
Programmer
Salesman

different behavior:
fixed salary
hourly salary
base salary plus commission
```

## 8. My Learning Notes

The main learning path is not just "learn more syntax".

It is:

```text
how to turn a problem into control flow;
how to choose the right data structure;
how to remove repeated code with functions;
how to pass behavior as data with higher-order functions;
how to add reusable behavior with decorators;
how to model larger problems with classes and objects.
```

A useful way to review:

```text
For each syntax point, ask what problem it solves.
For each example, ask what general pattern hides behind it.
For each longer program, ask what data moves through the program.
```

Examples:

```text
Prime numbers:
branch + loop + divisibility check

Dice statistics:
loop + random + list counters

Lottery:
random sampling + sorting + formatting

Decorator:
function object + wrapper + extra behavior

class modeling + list operations + sorting + magic methods

inheritance + abstract method + polymorphism
```
