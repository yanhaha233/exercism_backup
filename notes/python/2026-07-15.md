# Python Learning Notes - 2026-07-15

> Recent learning summary from Codewars practice: frequency counting, dictionaries, sets, `Counter`, and turning small kata patterns into project thinking.

## 1. Frequency counting with `dict`

Many beginner-to-intermediate kata are really counting problems.

The common pattern is:

```text
input data
-> count how many times each value appears
-> calculate an answer from those counts
```

Manual dictionary counting:

```python
counts = {}

for item in items:
    counts[item] = counts.get(item, 0) + 1
```

`dict.get(key, default)` is useful because it avoids writing an extra `if key in dict` check.

These two forms mean almost the same thing:

```python
if color in counts:
    counts[color] += 1
else:
    counts[color] = 1
```

```python
counts[color] = counts.get(color, 0) + 1
```

## 2. Pair counting: gloves kata

For the gloves problem, each color can form:

```python
count // 2
```

pairs.

Example:

```python
gloves = ["red", "green", "red", "blue", "blue"]
```

Counts:

```text
red: 2
green: 1
blue: 2
```

Result:

```python
2 // 2 + 1 // 2 + 2 // 2
# 1 + 0 + 1 = 2
```

Dictionary solution:

```python
def number_of_pairs(gloves):
    counts = {}

    for color in gloves:
        counts[color] = counts.get(color, 0) + 1

    return sum(count // 2 for count in counts.values())
```

Short solution using `set()` and `count()`:

```python
def number_of_pairs(gloves):
    return sum(gloves.count(color) // 2 for color in set(gloves))
```

This version is concise and readable for small kata input.

Performance note:

```python
gloves.count(color)
```

scans the whole list each time. If there are many unique colors, this repeats work. The dictionary or `Counter` version is better for larger data.

## 3. `set()` for unique values

`set()` removes duplicates:

```python
colors = ["red", "green", "red", "blue", "blue"]
set(colors)
# {"red", "green", "blue"}
```

This is useful when the calculation only needs to process each distinct value once.

Example:

```python
for color in set(gloves):
    pairs = gloves.count(color) // 2
```

## 4. `//` integer division

`//` means floor division.

For positive integers, it keeps the whole-number part:

```python
5 // 2
# 2
```

In pair-counting problems:

```python
count // 2
```

means "how many full pairs can be formed".

Examples:

```python
0 // 2  # 0
1 // 2  # 0
2 // 2  # 1
3 // 2  # 1
4 // 2  # 2
```

## 5. `collections.Counter`

`Counter` is the standard-library tool for frequency counting.

```python
from collections import Counter

counts = Counter(gloves)
```

Example:

```python
from collections import Counter

def number_of_pairs(gloves):
    return sum(count // 2 for count in Counter(gloves).values())
```

Use manual `dict` counting when practicing fundamentals. Use `Counter` when the problem is clearly a frequency-counting problem and you want cleaner code.

## 6. Missing Alphabets kata

This kata is also a frequency-counting problem.

The idea:

```text
1. Count each letter.
2. Find the maximum count.
3. Treat that maximum as the number of complete alphabet sets needed.
4. For each letter from a to z, output how many copies are missing.
```

Example:

```python
s = "aabbccddeeffgghhiijjkkllmmnnooppqqrrssttuuvvwwxxyy"
```

Most letters appear `2` times, but `z` appears `0` times.

So the missing result is:

```python
"zz"
```

Manual dictionary solution:

```python
def missing_alphabets(s):
    alphabet = "abcdefghijklmnopqrstuvwxyz"
    counts = {}

    for char in s:
        counts[char] = counts.get(char, 0) + 1

    max_count = max(counts.values())

    result = ""
    for char in alphabet:
        missing_count = max_count - counts.get(char, 0)
        result += char * missing_count

    return result
```

Cleaner version with `Counter`:

```python
from collections import Counter
import string

def missing_alphabets(s):
    counts = Counter(s)
    max_count = max(counts.values())

    return "".join(
        char * (max_count - counts.get(char, 0))
        for char in string.ascii_lowercase
    )
```

## 7. String repetition

Strings can be multiplied by integers:

```python
"z" * 2
# "zz"
```

This is useful when a missing value must be repeated.

Example:

```python
char = "z"
missing_count = 2

char * missing_count
# "zz"
```

## 8. `join()` with generated strings

When building a final string from many small strings, prefer:

```python
"".join(parts)
```

over repeated string concatenation in larger programs.

Example:

```python
result = "".join(
    char * missing_count
    for char in alphabet
)
```

In small kata, `result += ...` is acceptable and often easier to understand while learning.

## 9. Shared problem-solving pattern

The gloves kata and Missing Alphabets kata share the same first step:

```text
count frequencies
```

Then they differ in the final calculation:

```text
Gloves:
count // 2

Missing Alphabets:
max_count - current_count
```

This is an important learning point:

```text
Different stories can hide the same data pattern.
```

When reading a new kata, ask:

```text
Do I need to count things?
Do I need unique values?
Do I need the maximum or minimum count?
Do I need to compare each item with an expected amount?
```

## 10. From kata to small projects

Codewars usually gives a clear function:

```text
input -> output
```

Small projects are less clear:

```text
unclear need
-> split into features
-> decide what data to store
-> handle invalid input
-> save and reload data
```

A good transition project is a command-line TODO app.

Minimum requirements:

```text
Add a task
List tasks
Mark a task as done
Delete a task
Save tasks to a JSON file
Load tasks when the program starts
```

This practices the same basic tools:

```text
list
dict
loop
function
file reading and writing
simple data validation
```

The key idea:

```text
Do not wait until all requirements are clear.
Start with the smallest useful version, then let new requirements appear while building.
```
