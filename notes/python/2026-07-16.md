# Python Learning Notes - 2026-07-16

> Recent learning summary: Python decorators, `wrapper`, `return wrapper`, and how to refine a mental model by explaining code in my own words.

## 1. Basic decorator shape

A simple decorator usually has this structure:

```python
def record_time(func):
    def wrapper(*args, **kwargs):
        result = func(*args, **kwargs)
        return result

    return wrapper
```

Important roles:

```text
func
the original function being decorated

wrapper
the new function returned by the decorator

*args
all positional arguments

**kwargs
all keyword arguments, such as a=1 or b=2
```

The purpose of `*args` and `**kwargs` is to let `wrapper` accept almost any argument shape that the original function may need.

## 2. What `@record_time` means

This code:

```python
@record_time
def hello(name):
    print(f"hello {name}")
```

is equivalent to:

```python
def hello(name):
    print(f"hello {name}")

hello = record_time(hello)
```

The key step is:

```python
hello = record_time(hello)
```

The original `hello` function is passed into `record_time` as `func`.

Then `record_time` returns `wrapper`:

```python
return wrapper
```

So after decoration, the name `hello` points to `wrapper`, not directly to the original function.

## 3. `return wrapper`

`return wrapper` means:

```text
return the wrapper function object itself
```

It does not mean:

```text
run wrapper immediately
```

This distinction is important:

```python
return wrapper
```

returns the function so it can be called later.

```python
return wrapper()
```

would call `wrapper` immediately and return the result of that call. That is usually wrong for a decorator because the decorated function should run only when the user calls it.

## 4. The call chain after decoration

After decoration:

```python
hello = record_time(hello)
```

the name `hello` now refers to `wrapper`.

So when I call:

```python
hello("Alice")
```

the actual call chain is:

```text
hello("Alice")
-> wrapper("Alice")
   -> func("Alice")
      -> original hello("Alice")
```

A precise way to say it:

```text
After a function is decorated, the function name is rebound to the wrapper returned by the decorator.
Calling that function name enters wrapper first.
Then wrapper calls the original function through func.
```

## 5. Where the extra behavior goes

A decorator adds behavior around this line:

```python
result = func(*args, **kwargs)
```

Example with timing:

```python
import time
from functools import wraps

def record_time(func):
    @wraps(func)
    def wrapper(*args, **kwargs):
        start = time.time()

        result = func(*args, **kwargs)

        end = time.time()
        print(f"{func.__name__} took {end - start:.4f}s")

        return result

    return wrapper
```

The line:

```python
result = func(*args, **kwargs)
```

does two things:

```text
func(*args, **kwargs)
runs the original function

result
stores the return value of the original function
```

So `result` is not the execution process itself. It is the value produced after the original function finishes.

## 6. Why `return result` is needed

If the original function returns a value, the wrapper should usually return that value too.

Example:

```python
def add(a, b):
    return a + b
```

After decoration, calling `add(1, 2)` is really calling `wrapper(1, 2)`.

So `wrapper` must return the original result:

```python
result = func(*args, **kwargs)
return result
```

Otherwise, the decorated function may lose its original return value and return `None`.

## 7. `functools.wraps`

When writing real decorators, use:

```python
from functools import wraps
```

and:

```python
@wraps(func)
def wrapper(*args, **kwargs):
    ...
```

This preserves useful metadata from the original function, such as:

```text
function name
docstring
module information
```

Without `wraps`, Python may think the decorated function is named `wrapper`.

## 8. Thinking review

My original understanding was mostly correct:

```text
The decorator receives the function to be decorated.
wrapper is where the decorator behavior is implemented.
*args and **kwargs are used because the decorator does not know the original function's parameters in advance.
Extra behavior can be placed before or after calling func(*args, **kwargs).
return result preserves the original function's return value.
return wrapper gives the decorated name a new function to call.
```

The wording that needed refinement:

```text
Not precise:
result is the execution process of the decorated function.

More precise:
func(*args, **kwargs) is the execution process.
result is the return value produced by that execution.
```

Another refinement:

```text
Not precise:
The decorated function finally returns the decorator.

More precise:
The decorated function name is rebound to the wrapper function returned by the decorator.
```

This is a good learning method:

```text
1. First explain the code in my own words.
2. Then translate the @ syntax into the equivalent assignment.
3. Separate function objects from function calls.
4. Separate execution from return values.
5. Improve the wording until the mental model matches what Python actually does.
```

Final mental model:

```text
A decorator is a function that receives a function and returns a new function.
The returned wrapper replaces the original function name.
When that name is called, wrapper runs first, and wrapper can call the original function through func.
```
